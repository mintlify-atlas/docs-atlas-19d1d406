## Overview

The Darwin Dashboard is a Next.js application that provides a visual interface for running agents, monitoring sessions, and viewing analytics.

<Info>
The dashboard communicates with the Express API (port 3002) to control agents and stream results in real-time.
</Info>

## Starting the Dashboard

### With API Server

From the **root directory**:

```bash
npm run start:services
```

This starts both:
- **Express API** on http://localhost:3002
- **Dashboard** on http://localhost:3001

### Dashboard Only

From `packages/darwin-sdk/dashboard`:

```bash
cd packages/darwin-sdk/dashboard
npm run dev
```

<Warning>
The dashboard requires the API server to be running on port 3002. Make sure to start it first.
</Warning>

### Run Agent Form

Create and launch new agents:

**Fields**:
- **Website URL** - Target website
- **Task Description** - Natural language task
- **AI Model** - Choose from available models
- **Max Steps** - Limit agent actions
- **Environment** - LOCAL or BROWSERBASE

**Example**:

```
Website: http://localhost:3000
Task: Sign up for an account and complete onboarding
Model: google/gemini-3-flash-preview
Max Steps: 30
Environment: LOCAL

[Run Agent]
```

On submit, makes a request to:

```bash
POST http://localhost:3002/api/run
```

Returns a `sessionId` for tracking.

### Live Streaming

Sessions stream logs in real-time using Server-Sent Events:

```typescript
// In dashboard code
const eventSource = new EventSource(
  `http://localhost:3002/api/stream/${sessionId}`
);

eventSource.addEventListener('think', (e) => {
  const data = JSON.parse(e.data);
  addLog('💭', data.message);
});

eventSource.addEventListener('action', (e) => {
  const data = JSON.parse(e.data);
  addLog('🔧', data.message);
});

eventSource.addEventListener('status', (e) => {
  const data = JSON.parse(e.data);
  updateStatus(data.status);
});

eventSource.addEventListener('result', (e) => {
  const data = JSON.parse(e.data);
  showResult(data);
  eventSource.close();
});
```

## API Client

The dashboard uses a centralized API client:

```typescript
// lib/api-client.ts
const API_BASE = 'http://localhost:3002/api';

export const apiClient = {
  async getStatus() {
    const res = await fetch(`${API_BASE}/status`);
    return res.json();
  },
  
  async getConfig() {
    const res = await fetch(`${API_BASE}/config`);
    return res.json();
  },
  
  async runAgent(config) {
    const res = await fetch(`${API_BASE}/run`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(config)
    });
    return res.json();
  },
  
  async getSession(sessionId) {
    const res = await fetch(`${API_BASE}/session/${sessionId}`);
    return res.json();
  },
  
  async getSessions() {
    const res = await fetch(`${API_BASE}/sessions`);
    return res.json();
  },
  
  streamSession(sessionId, callbacks) {
    const eventSource = new EventSource(`${API_BASE}/stream/${sessionId}`);
    
    Object.entries(callbacks).forEach(([event, handler]) => {
      eventSource.addEventListener(event, (e) => {
        handler(JSON.parse(e.data));
      });
    });
    
    return eventSource;
  }
};
```

## Deployment

### Build for Production

```bash
cd packages/darwin-sdk/dashboard
npm run build
npm run start
```

Runs on port 3001 by default.

### Environment Variables

```bash .env.local
NEXT_PUBLIC_API_URL=http://localhost:3002
```

Update `NEXT_PUBLIC_API_URL` to point to your API server.

### Docker

```dockerfile
FROM node:20-alpine AS base

WORKDIR /app
COPY package*.json ./
RUN npm install

COPY . .
RUN npm run build

EXPOSE 3001
CMD ["npm", "run", "start"]
```

```bash
docker build -t darwin-dashboard .
docker run -p 3001:3001 darwin-dashboard
```

## Troubleshooting

### Dashboard shows "API Offline"

- Check that Express API is running on port 3002
- Verify `NEXT_PUBLIC_API_URL` is correct
- Check for CORS errors in browser console

### Sessions not streaming

- Ensure SSE is not blocked by proxy/firewall
- Check browser DevTools Network tab for EventSource connection
- Verify API endpoint returns `Content-Type: text/event-stream`

### Metrics not updating

- Refresh the page
- Check `/api/metrics` endpoint directly
- Verify sessions are being tracked correctly
